    Function addDict

    • addDict(
          state: TypeCheckerState,
          name: string,
          dict: DictTerm,
      ): Result<TypingError, TypeCheckerState>

      Adds a dictionary value to the typing context (Γ) under a given name.

      A dictionary is a term of the form:

      dict Trait<Type> { method₁ = impl₁, ... }

      and represents explicit evidence that a type implements a trait.

      Unlike addTraitImpl, which registers a global trait instance, addDict introduces a local dictionary variable, similar to:

      d : Dict<Eq, Int>

      This is useful in:

      • REPL environments
      • manual dictionary passing
      • intermediate testing
      • debugging trait logic

      What this function does

      1. Infers the type of the dictionary using inferDictType. This ensures:

        • the trait exists
        • all required methods are present
        • method implementations are correctly typed
        • the dictionary type is well‑formed

      2. Creates a DictBinding:

        name : Dict<trait, type>

      3. Inserts this binding into the context using addBinding.

      The full dictionary term itself remains available in state.ctx under the dict property of the binding.

      When to use

      • When the user explicitly writes a dictionary value they want to name:

        let eqInt = dict Eq<Int> { eq = ... }

      • When testing trait implementations manually:

        addDict(state, "eqInt", eqIntDict);

      • When manually passing dictionaries during trait applications (without relying on auto‑instantiation).

      Error cases

      • Any error produced by inferDictType
      • duplicate_binding if the dictionary name already exists

      Parameters

      • state: TypeCheckerState

        Current typechecker state

      • name: string

        Variable name for the dictionary (e.g., "eqInt")

      • dict: DictTerm

        The dictionary term to bind

      Returns Result<TypingError, TypeCheckerState>

      Updated state or a TypingError

      Example: Adding a dictionary:

      const dict = dictTerm("Eq", Int, [
        ["eq", lamTerm("x", Int, lamTerm("y", Int, conTerm("true", Bool)))]
      ]);

      addDict(state, "eqInt", dict);
      // => adds eqInt : Dict<Eq, Int>

      Example: Using the dictionary:

      traitMethodTerm(varTerm("eqInt"), "eq");
      // accesses the eq method inside eqInt
