    Function unifyKinds

    • unifyKinds(left: Kind, right: Kind): Result<TypingError, null>

      Unifies two kinds, producing a kind_mismatch error if they differ.

      What it checks:

      • Kind equality is purely structural:
        • * unifies with *
        • * → * unifies with * → * (recursively unifying input and output kinds)
      • Uses kindsEqual for the structural comparison.

      Why it matters:

      Kind unification is the kind‑level analogue of type unification. It ensures type constructors are used correctly:

      • List :: * → *
      • List<Int> :: *
      • Int<Int> → error (Int :: *, not * → *)

      This function is used by:

      • the constraint solver (processConstraint)
      • type application kind checking (checkAppKind)
      • polymorphic binder validation (checkForallKind, checkLamKind)

      Errors produced:

      • kind_mismatch: kinds differ structurally

      Parameters

      • left: Kind

        The first kind

      • right: Kind

        The second kind

      Returns Result<TypingError, null>

      ok(null) if equal, otherwise kind_mismatch

      Example

      unifyKinds(starKind, starKind);                  // ok
      unifyKinds(arrowKind(starKind, starKind),
                 arrowKind(starKind, starKind));       // ok

      unifyKinds(starKind, arrowKind(starKind, starKind));
      // err({ kind_mismatch: ... })
