    Function isAssignableTo

    • isAssignableTo(state: TypeCheckerState, from: Type, to: Type): boolean

      Checks whether a type from can be assigned to a type to (a lightweight, fast version of subtyping).

      What it does:

      • Handles the two special rules for bottom (⊥):
        • ⊥ is assignable to any type (⊥ <: T)
        • No type is assignable to ⊥ unless it is also ⊥
      • Otherwise, falls back to structural type equality after normalization.

      Why this exists:

      • Many checks only need a simple notion of compatibility:
        • pattern checking
        • tuple/record element comparisons
        • constructor payload validation
      • Full subtyping (subsumes) is more expensive and handles polymorphism, so this function provides a quick path when that power isn’t required.

      Related:

      • isBottom — detects ⊥
      • typesEqual — structural equality used as the final check
      • subsumes — the full subtyping engine (more general)

      Parameters

      • state: TypeCheckerState

        Typechecker state used for normalization and bottom detection

      • from: Type

        The source type (value's type)

      • to: Type

        The target type (expected type)

      Returns boolean

      true if from is assignable to to, else false

      Example: Bottom:

      isAssignableTo(state, neverType, conType("Int"));  // true
      isAssignableTo(state, conType("Int"), neverType);  // false

      Example: Regular assignment:

      isAssignableTo(state, conType("Bool"), conType("Bool")); // true
      isAssignableTo(state, conType("Int"),  conType("Bool")); // false
