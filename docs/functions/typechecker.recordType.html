    Function recordType

    • recordType(record: [string, Type][]): { record: [string, Type][] }

      Constructs a record type:

      { label₁: T₁, label₂: T₂, ... }

      Record types describe structural products with named fields. They support width subtyping:

      { x: Int }  <:  { x: Int, y: Bool }

      meaning a record with more fields can be used where a record with fewer fields is expected.

      Examples:

      • { x: Int, y: Bool }
      • { name: String, age: Int }

      Used in:

      • record terms (recordTerm)
      • pattern matching (recordPattern)
      • projection (projectTerm)
      • structural unification (unifyRecordTypes)
      • normalization and type equality

      Kind checking:

      • Every field type must have kind * (validated by checkRecordKind)

      Parameters

      • record: [string, Type][]

        An array of [label, type] pairs describing fields

      Returns { record: [string, Type][] }

      A RecordType node { record: [...] }

      Example

      recordType([
        ["x", conType("Int")],
        ["y", conType("Bool")]
      ]);
      // => { x: Int, y: Bool }

      Example: Width subtyping:

      const small = recordType([["x", Int]]);
      const big   = recordType([["x", Int], ["y", Bool]]);

      isAssignableTo(state, small, big);  // true
