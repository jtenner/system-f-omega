    Function addTerm

    • addTerm(
          state: TypeCheckerState,
          name: string,
          term: Term,
      ): Result<TypingError, TypeCheckerState>

      Adds a new term binding to a typechecker state.

      This is used for:

      • defining global terms in a module
      • adding local bindings in a REPL or top‑level environment
      • programmatically extending a context with a value definition

      The steps performed:

      1. Infer the type of the provided term (inferType).
      2. On success, build a TermBinding: name : inferredType
      3. Insert the binding into the context using addBinding.

      If the term fails to typecheck, the error is returned unchanged.

      Error cases

      • Any error produced by inferType
      • duplicate_binding (raised inside addBinding if the name already exists in the context, unless overrides are permitted)

      Parameters

      • state: TypeCheckerState

        Current typechecker state (context + meta environment)

      • name: string

        The term variable name to bind

      • term: Term

        The term whose type should be inferred and added

      Returns Result<TypingError, TypeCheckerState>

      An updated TypeCheckerState or a TypingError

      Example: Add a simple value:

      addTerm(state, "x", conTerm("42", conType("Int")));
      // adds: x : Int

      Example: Add a function:

      addTerm(state, "id", lamTerm("x", Int, varTerm("x")));
      // adds: id : Int → Int

      • inferType Determines the term’s type
      • termBinding Constructs the context binding
      • addBinding Inserts the binding into the context
