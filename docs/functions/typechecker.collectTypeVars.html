    Function collectTypeVars

    • collectTypeVars(type: Type, vars?: Set<string>): string[]

      Collects all free type variable names that appear inside a type.

      This function walks the structure of a type and gathers every occurrence of a VarType ({ var: "A" }). It excludes variables that are bound by:

      • forall (universal quantification)
      • lam (type‑level lambda abstraction)
      • mu (recursive type binder)

      In other words, it returns the set of free type variables in the type.

      Why this matters

      Free type variables are used for:

      • Dependency analysis during module importing (e.g., which type constructors a type depends on)
      • Generalization checks (e.g., only generalize variables not in scope)
      • Renaming / alpha‑conversion tasks
      • Pretty‑printing and debugging

      This function does not detect meta‑variables (?N). For that, see getUnboundMetas.

      Bound vs free example

      const t = forallType("A", starKind,
        arrowType(varType("A"), varType("B"))
      );

      collectTypeVars(t)
      // => ["B"]   (A is bound, B is free)

      What it walks

      Recurses through:

      • type applications (F<A>)
      • arrow types
      • tuples
      • records
      • variants

      Skips substitution under binders:

      • forall A. ...
      • lam A. ...
      • mu A. ...

      Parameters

      • type: Type

        The type to inspect

      • vars: Set<string> = ...

        Internal accumulator (optional)

      Returns string[]

      All free type variable names as an array

      Example: Simple

      collectTypeVars(arrowType(varType("A"), varType("B")));
      // => ["A", "B"]

      Example: Bound variable (not collected)

      collectTypeVars(forallType("A", starKind, varType("A")));
      // => []

      Example: Records and variants

      collectTypeVars(recordType([["x", varType("A")], ["y", varType("B")]]));
      // => ["A", "B"]
