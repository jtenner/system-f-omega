    FieldScheme: Type

    A field scheme inside an enum (ADT) variant — simply a Type with unbound type parameters.

    What it represents FieldScheme describes the payload type of a variant before substituting the enum’s type parameters. It is used inside EnumDef.variants.

    For example, in:

    enum Option<T> {
      None : ()
      Some : T
    }

    the field schemes are:

    • () for None
    • T for Some

    These schemes may include type parameters that will be instantiated when the enum is applied to concrete arguments:

    Option<Int>.Some → Int

    Why it's useful Field schemes allow enum definitions to:

    • be parametric (using unbound vars like T, A, E)
    • support normalization into VariantType via normalizeType
    • participate in recursive definitions (List<T> has Cons : (T, List<T>))

    The typechecker uses them in:

    • addEnum — validating that each scheme has kind *
    • inferInjectType — matching constructor payloads
    • normalizeType — substituting params with actual type arguments

    Examples

    A simple payload:

    import { varType } from "system-f-omega";

    const someField: FieldScheme = varType("T");  // T

    A tuple payload for a Cons constructor:

    import { tupleType, varType, appType, conType } from "system-f-omega";

    const consField: FieldScheme = tupleType([
      varType("T"),                     // head
      appType(conType("List"), varType("T"))  // tail
    ]);

    Unit payload:

    import { tupleType } from "system-f-omega";

    const noneField: FieldScheme = tupleType([]);  // ()
