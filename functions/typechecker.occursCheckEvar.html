    Function occursCheckEvar

    • occursCheckEvar(env: MetaEnv, evar: string, ty: Type): boolean

      Performs the occurs check for a meta‑variable.

      What it checks:

      • Returns true if meta‑variable evar appears anywhere inside type ty.
      • Follows solved meta‑vars recursively (e.g., ?1 := ?0, so checking ?0 inside ?1 succeeds).

      Why it matters:

      • Prevents assigning a meta‑variable a type that contains itself, which would create an infinite type such as:
        • ?0 := (?0 → Int)
      • Used by solveMetaVar to reject cyclic unification.

      How it recurses:

      • Walks all type constructors: arrows, apps, records, variants, tuples, μ‑types.
      • Ignores regular type variables (they are not meta‑vars).

      Parameters

      • env: MetaEnv

        Meta‑environment containing current meta‑var solutions

      • evar: string

        Meta‑variable name to search for ("?0")

      • ty: Type

        Type to scan for occurrences of evar

      Returns boolean

      true if evar occurs in ty, otherwise false

      Example

      // ?0 appears inside (?0 → Int)
      const ty = arrowType({ evar: "?0" }, conType("Int"));
      occursCheckEvar(env, "?0", ty);  // true

      Example

      // Following solved meta-vars:
      env.solutions.set("?1", { evar: "?0" });
      occursCheckEvar(env, "?0", { evar: "?1" });  // true
